# Chat Persistence with User Authentication - Implementation Guide

## Overview
Your AgriSense application has a complete chat persistence system that:
- ✅ Saves RAG assistant conversations to MongoDB
- ✅ Associates chats with authenticated users (JWT tokens)
- ✅ Prevents unauthorized access to other users' chats
- ✅ Allows users to retrieve and continue past conversations
- ✅ Automatically saves after each AI response (for logged-in users only)

## Architecture

### 1. Frontend Flow (React/Next.js)
**Location**: `frontend/src/features/assistant/AssistantContent.jsx`

```jsx
// User sends message → RAG API processes → AI response received
const response = await askRagAssistant({ question, language, chatHistory });

// Automatically save to MongoDB if user is logged in
await persistMessages(nextMessages);

// persistMessages function:
const persistMessages = async (nextMessages) => {
  if (!token) return;  // ⚠️ No save for unauthenticated users
  try {
    const data = await saveChatSession({
      token,        // JWT token for authentication
      sessionId,    // Existing session ID (for updates) or null (for new)
      title,        // Auto-generated from first user message
      language,     // Current conversation language
      messages,     // All messages in conversation
    });
    if (data.session?.id) {
      setSessionId(data.session.id);  // Store for future updates
    }
  } catch (err) {
    console.error("Failed to save chat session:", err);
  }
};
```

**Location**: `backend/src/routes/chatHistory.js`

Creates new chat session or updates existing one
```javascript

  language: "en",
  messages: [
    {
      sender: "user",
      text: "What crop for summer?",
      language: "en",

  success: true,

    id: "new-session-id",      // ← Generated by MongoDB

    title: "Summer Crop Selection",

    updatedAt: "2026-05-16T..."
  }

```

### 3. MongoDB Schema
**Collection**: `chatSessions`

```javascript
{
  _id: ObjectId,              // MongoDB auto-generated
  userId: String,             // Extracted from JWT token
  title: String,              // First user message (truncated to 80 chars)
  language: String,           // 'en', 'ur', 'pa'
  messages: [
    {
      sender: String,         // 'user' or 'ai'
      text: String,
      language: String,
      sources: Array,         // RAG document sources
      createdAt: Date
    }
  ],
  createdAt: Date,
  updatedAt: Date
}
```

### 4. Authentication Middleware
**Location**: `backend/src/middleware/auth.js`

The middleware:
1. Extracts JWT token from `Authorization: Bearer <TOKEN>` header
2. Verifies token signature
3. Extracts `userId` from token payload
4. Attaches to request: `req.user = { userId, ... }`

```javascript
const token = authHeader.substring(7);  // Remove 'Bearer '
const decoded = jwt.verify(token, process.env.JWT_SECRET);
req.user = decoded;
next();
```

### 5. RAG System Integration
**Location**: `rag_system/app.py`

The RAG API (`/ask` endpoint) provides:
- `answer`: Text response from LLM
- `language`: Detected/requested language
- `sources`: Array of document references
  ```javascript
  {
    source: "filename.pdf",
    page: 5,
    chunk: 2
  }
  ```

These are captured by the frontend and included in chat messages.

## Security Features

### 1. User Isolation
```javascript
// Backend ONLY returns sessions for authenticated user
db.collection('chatSessions').find({ userId: req.user.userId })
// Even with valid token, user can only access OWN sessions
```

### 2. JWT Token Validation
```javascript
// Every chat-history request requires valid JWT
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc...
// Token expires in app.js (typically 7 days)
```

### 3. Unauthenticated Chat (No Persistence)
```javascript
// Frontend checks:
const persistMessages = async (nextMessages) => {
  if (!token) return;  // ✅ No DB write for anonymous users
  // ...
};
```

### 4. Message Validation
```javascript
// Backend validates before saving
if (!Array.isArray(messages) || messages.length === 0) {
  return res.status(400).json({ message: 'Messages are required' });
}
```

## Troubleshooting

### "No token provided. Please login."
- **Cause**: Frontend didn't include JWT in request header
- **Fix**: User must click "Login to save" button first

### "Chat session not found"
- **Cause**: User trying to access another user's session
- **Fix**: Authorization check working correctly. This is expected behavior.

### Messages not saving
- **Cause**: User is not logged in
- **Fix**: Click "Login to save" button, then continue chatting

### Empty "Saved Chats" after login
- **Cause**: First time login (no previous chats saved)
- **Fix**: Chat and wait for 2 AI responses to be asked to log in, then refresh

### MongoDB connection error
- **Cause**: MongoDB URI invalid or cluster paused
- **Fix**: Check `.env` file for `MONGODB_URI` and verify cluster is running
